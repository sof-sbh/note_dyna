# 10. Corrections et points de vigilance

Les notes initiales ont été nettoyées et corrigées sur les points suivants :

| Formulation initiale | Correction |
|---|---|
| GrailDB = Graph DB | Grail est une plateforme de données unifiée et contextuelle, pas simplement une base graphe. |
| Guardian déclenche toujours des actions correctives | Guardian évalue des objectifs de qualité et peut être relié à des workflows de remédiation. |
| MTDR = Mean Time To Repair | Le terme le plus courant est **MTTR**. MTTR peut signifier Repair, Recover ou Restore selon le contexte. |
| RPO = Recovery Time Objective | RPO signifie **Recovery Point Objective**. |
| RTO = Recovery Time Point | RTO signifie **Recovery Time Objective**. |
| Full-Stack implique automatiquement tous les logs | La collecte des logs dépend de la configuration et des règles de collecte. |
| ActiveGate toujours entre OneAgent et Dynatrace | ActiveGate est requis dans certains scénarios, mais pas systématiquement. |
| Problem = groupe d’events uniquement | C’est une bonne simplification, mais Dynatrace ajoute corrélation, contexte et causalité. |
