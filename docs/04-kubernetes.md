# 4. Kubernetes

| Élément | Rôle |
|---|---|
| **Cluster** | Ensemble de ressources Kubernetes administrées comme une plateforme. |
| **Node** | Machine virtuelle ou physique exécutant les workloads. |
| **Namespace** | Isolation logique des ressources. |
| **Pod** | Plus petite unité d’exécution Kubernetes. |
| **Workload** | Objet pilotant des pods : Deployment, StatefulSet, DaemonSet, Job, etc. |
| **Cluster Explorer** | Vue Dynatrace consacrée à l’état et aux dépendances du cluster. |
| **Dynatrace Operator** | Automatise le déploiement et le cycle de vie des composants Dynatrace. |

## Cloud Native Full-Stack

Ce mode vise une observabilité complète des environnements Kubernetes :

- cluster et nœuds ;
- namespaces et workloads ;
- pods et conteneurs ;
- services ;
- traces distribuées ;
- logs selon configuration ;
- corrélation avec l’infrastructure et les applications.
